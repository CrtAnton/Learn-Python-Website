Lesson: Shell Environment

Welcome to the sixth lesson of your Linux learning journey! In this lesson, you'll dive into the shell environment and learn about environment variables, command history, and useful shortcuts to enhance your Linux experience.

1. Environment Variables

Environment variables are like settings that affect how your shell and programs work. They store information such as paths to important directories, configuration options, and more.

Example:

echo $HOME
echo $PATH

In this example, $HOME represents your home directory, and $PATH holds a list of directories where the shell searches for executable files.

2. Command History and Shortcuts

The shell keeps track of the commands you've typed. You can access your command history using the arrow keys or specific shortcuts.

Example:

history
!! (repeats the last command)
!5 (repeats command number 5)

The history command shows your command history. The !! shortcut repeats the last command, and !5 repeats the command numbered 5 in your history.

3. Aliases

Aliases are custom shortcuts for commands or command combinations. They help you create simpler and quicker ways to execute frequently used commands.

Example:

alias ll='ls -l'
alias update='sudo apt update && sudo apt upgrade'

In this example, the alias ll is a shorthand for ls -l, which lists files with detailed information. The alias update combines two commands to update and upgrade packages.

Exercises:

  1. Print the value of the $USER environment variable.
  2. Use the history command to view your recent command history.
  3. Create an alias called gd that navigates to your desktop directory.

Solutions:

  1. echo $USER
  2. history
  3. alias gd='cd ~/Desktop'
