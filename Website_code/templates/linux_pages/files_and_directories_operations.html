Lesson: File and Directory Operations

Welcome to the third lesson of your Linux learning journey! In this lesson, you'll explore essential file and directory operations that are fundamental for navigating and managing your system effectively.

1. Understand File and Directory Structure

Linux organizes its file system hierarchically, similar to the branches of a tree. At the very top is the root directory ("/"), from which all other directories and files stem. Directories can contain subdirectories and files, creating a structured and organized system.

Example:


    /
    ├── home
    │   ├── user
    │   │   ├── documents
    │   │   └── pictures
    │   └── guest
    │       └── downloads
    ├── var
    ├── etc
    ├── usr
    └── ...

2. Use Relative and Absolute Paths

Paths are like addresses that help you navigate the file system. Absolute paths start from the root directory, while relative paths start from your current location. Mastering both types is essential for efficient navigation.

Example:


    Absolute: /home/user/documents
    Relative: documents

3. cat, less, more - View File Contents

These commands allow you to view the contents of a file. cat displays the entire file at once, less and more offer pagination and navigation.

Example:

cat myfile.txt
less largefile.txt
more readme.txt

In the example above, less and more commands allow you to scroll through the file content. Use the "Space" key to move forward and "B" key to move backward.

4. head, tail - Display File Beginning or End

head displays the start, while tail displays the end of a file. Useful for quick content checks.

Example:

head -n 10 log.txt
tail -n 20 biglog.txt

The flag -n specifies the number of lines to display. In the above examples, the first 10 lines of "log.txt" and the last 20 lines of "biglog.txt" are shown.

5. grep - Search for Text Within Files

grep helps find text patterns within files. It shows matching lines, helping you quickly identify relevant information.

Example:

grep "error" logfile.txt
grep -r "searchterm" /path/to/search

The flag -r enables recursive searching, allowing you to search within subdirectories. In the second example, grep searches for "searchterm" within the specified path.

6. find - Search for Files and Directories

find searches for files and directories based on criteria like name, type, size, etc. It's a versatile tool for locating items.

Example:

find /home/user -name "*.txt"
find /var/log -size +1M

In the first example, -name "*.txt" filters files with a ".txt" extension. The second example searches for files larger than 1 megabyte in size.

7. chmod - Change File Permissions

chmod adjusts file permissions that control read, write, and execute access for owners, groups, and others.

Example:

chmod u+r myfile.txt
chmod 755 script.sh

In the first example, u+r adds read permission for the owner. In the second example, 755 grants read, write, and execute permissions to the owner, and read and execute permissions to the group and others.

8. chown - Change File Ownership

chown changes the ownership of files and directories. Ownership affects control and modification rights.

Example:

chown user:group myfile.txt
chown -R user:group myfolder

The first example changes ownership to a specific user and group. The -R flag is used to perform recursive ownership change on a directory and its contents.

Exercises:

  1. Create a directory structure: "root" containing "documents" and "pictures," and "documents" containing "work" and "personal" subdirectories.
  2. Use both relative and absolute paths to navigate to the "work" subdirectory.
  3. View the contents of a file named "report.txt" using cat, less, and more.
  4. Display the first 10 lines of the file "log.txt" using head.
  5. Search for the word "error" in all text files within the "logs" directory using grep.
  6. Find all files larger than 1MB in the system using find.
  7. Change the permissions of a file named "data.txt" to allow read and write access to the owner using chmod.
  8. Change the ownership of a directory "shared" to a user named "team" using chown.

Solutions:

  1. mkdir root, mkdir root/documents, mkdir root/pictures, mkdir root/documents/work, mkdir root/documents/personal
  2. cd root/documents/work
  3. cat report.txt, less report.txt, more report.txt
  4. head -n 10 log.txt
  5. grep "error" logs/*.txt
  6. find / -type f -size +1M
  7. chmod u+rw data.txt
  8. chown team shared
