Lesson: System and Networking

Welcome to the seventh lesson of your Linux learning journey! In this lesson, you'll explore tools and commands that provide insights into your system's performance and networking capabilities.

1. ps - View Running Processes

The ps command displays information about processes running on your system. It's a useful way to see what programs are currently active.

Example:

ps aux
ps -ef

In these examples, the aux and -ef options provide detailed information about all running processes.

2. top, htop - Monitor System Resources

The top and htop commands show real-time information about system resource usage, including CPU, memory, and more.

Example:

top
htop

Both top and htop display a list of processes with resource usage details, updating in real-time.

3. df, du - Disk Usage and Space Analysis

The df command shows information about disk space usage, while the du command helps you analyze directory space consumption.

Example:

df -h
du -sh /home/user

The -h flag in df displays sizes in human-readable format (e.g., KB, MB). The -sh flag in du summarizes and displays the size of the specified directory.

4. ping - Check Network Connectivity

The ping command tests network connectivity by sending packets to a remote host and receiving responses.

Example:

ping www.google.com

In this example, www.google.com is the hostname being pinged. The command provides information about the round-trip time and packet loss.

5. ifconfig or ip - Network Configuration

The ifconfig and ip commands provide information about network interfaces and their configurations.

Example:

ifconfig
ip addr show

Both commands display details about network interfaces, IP addresses, and more.

6. netstat, ss - Network Statistics

The netstat and ss commands offer insights into network statistics, including open ports and active connections.

Example:

netstat -tuln
ss -tuln

The -tuln options show TCP ports in use and listening, both for netstat and ss.

Exercises:

  1. Use ps to list all running processes on your system.
  2. Monitor system resources using the top or htop command.
  3. Check the disk space usage of your home directory with df.
  4. Ping a website or server of your choice to test network connectivity.
  5. Use ifconfig or ip to display details of your network interfaces.
  6. List the open ports on your system using netstat or ss.

Solutions:

  1. ps aux
  2. top or htop
  3. df -h
  4. ping www.google.com
  5. ifconfig or ip addr show
  6. netstat -tuln or ss -tuln
