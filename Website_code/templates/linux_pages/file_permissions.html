Lesson: File Permissions and Security

Welcome to the ninth lesson of your Linux learning journey! In this lesson, you'll delve into file permissions and security measures, essential concepts to keep your files safe and organized.

1. Understanding the Permission System

In Linux, each file and directory has permissions that determine who can access, modify, or execute them. Permissions are divided into three groups: owner, group, and others. Each group can have three types of permissions: read, write, and execute.

Example:

ls -l
-rw-r--r-- 1 user group 1000 Aug 01 file.txt

In the above example, -rw-r--r-- represents the permissions for the owner, group, and others. The file's owner can read and write, while both the group and others can only read.

2. Using umask to Set Default Permissions

The umask command allows you to set default permissions for newly created files and directories. It works by subtracting the specified permissions from the maximum permissions possible.

Example:

umask 0022

In this example, the umask value of 0022 ensures that newly created files will have permissions of 644 and directories will have permissions of 755.

3. Secure File Copying (rsync)

The rsync command is a versatile tool for securely copying and synchronizing files and directories between different locations.

Example:

rsync -avz source/ destination/

In this example, the -avz flags maintain file permissions, copy directories recursively, and use compression for efficient data transfer.

Exercises:

  1. Create a file named secret.txt and set its permissions to be readable and writable only by you.
  2. Use umask to set default permissions that allow read, write, and execute for the owner, and read and execute for others.
  3. Copy a directory named documents from your home directory to a backup location using rsync.

Solutions:

  1. touch secret.txt
chmod 600 secret.txt
  2. umask 0007
  3. rsync -avz ~/documents/ /backup/documents/
