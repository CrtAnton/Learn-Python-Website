Functions in Python

Introduction to Functions

A function is a block of organized, reusable code that is used to perform a single, related action. Functions provide better modularity for your application and a high degree of code reusing.

Parameters vs. Arguments

While defining a function, we often require inputs to provide variability to our functions. These inputs are called parameters. When we call the function and provide values for these inputs, the values are termed as arguments.

Parameters are the names listed in the function definition. Arguments are the real values passed to the function when it is invoked.


def greet_user(name):  # 'name' is a parameter
    print(f"Hello, {name}!")

greet_user("Alice")  # 'Alice' is an argument

In the above example, name is a parameter of the function greet_user, while "Alice" is the argument passed to the function when it's called.

Functions: Returning vs. Non-returning

Returning Functions

A function that sends a result back to the line from which it was called is known as a returning function. The return keyword is used to return a value from the function.


    def add_numbers(a, b):
        return a + b
    
    result = add_numbers(3, 4)  # The returned value 7 is stored in the variable 'result'
    print(result)  # Outputs: 7

In the example above, the add_numbers function calculates the sum of its two parameters, a and b, and returns the result. When we call this function with arguments 3 and 4, it returns 7, which we store in the result variable and then print.

Non-returning Functions

Functions that perform an action but don't send a value back to the line from which they were called are termed as non-returning functions.


    def display_message(message):
        print(message)
    
    display_message("Hello, World!")  # Outputs: Hello, World!

In the above code, the function display_message simply prints out the provided message. It doesn't return any value back to the caller.
